# Get Brand

Fetches a single brand resource for the authenticated organization.

This endpoint is the direct resolver for `result_url: brands/{brand_id}` returned by completed brand extraction jobs.

## Endpoint

- `GET /api/v1/brands/:id`

## Authentication

```http
Authorization: Bearer <your_api_key>
```

## Path parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `id` | `uuid` | Yes | Brand ID to retrieve. |

## Query parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `expand` | `"latest_job"` | No | Include latest extraction job metadata as `data.expanded.latest_job`. |
| `include` | `"latest_job"` | No | Alias of `expand=latest_job`. |
| `fields` | `csv` | No | Select specific brand fields only. |

## Security behavior

- Returns `404` when:
  - the brand does not exist, or
  - the brand belongs to a different organization.

## Example cURL

```bash
curl -X GET "https://api.example.com/api/v1/brands/1c26bcfa-e50e-4800-a7df-5b97db5fd50c" \
  -H "Authorization: Bearer pi_live_***"
```

## Example cURL with expansion

```bash
curl -X GET "https://api.example.com/api/v1/brands/1c26bcfa-e50e-4800-a7df-5b97db5fd50c?include=latest_job&fields=id,name,brand_dna" \
  -H "Authorization: Bearer pi_live_***"
```

## When to use expansion and fields

- Use expansion when brand detail screens also show extraction status.
- Use fields when you only need a subset (e.g. `id,name`).

## When to not use expansion and fields

- Avoid expansion for high-throughput internal lookups that only need brand data.
- Avoid fields when your consumer expects the complete canonical brand object.
